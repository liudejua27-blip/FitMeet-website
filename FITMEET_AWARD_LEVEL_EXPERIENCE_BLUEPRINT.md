# FitMeet 获奖级企业官网体验蓝图（桌面端）

> 版本：2026-07-11  
> 范围：仅桌面端  
> 目标：以一条清晰、可感知、可落地的品牌叙事，统一 FitMeet 企业官网的页面结构、文案、视觉和动效。

---

## 1. 北极星命题

### 品牌世界观

**Social World**

世界很大，别一个人抵达。

### 产品承诺

**让社交更简单。**

### 产品解释

FitMeet 帮你把一个想见面的想法，整理成一份可以确认的计划。它辅助你开始，但不会替你决定。

### 全站叙事主线

1. **产生想法**：我想打球、散步、旅行，或者找人一起完成一件事。
2. **找到可能**：看见场景、同伴与真实发生过的瞬间。
3. **整理计划**：把时间、地点、人数、预算和边界说清楚。
4. **建立信任**：所有建议都可以修改，所有决定都由用户确认。
5. **走进现实**：把线上形成的计划带进真实生活。

### 唯一品牌动效

全站只使用一套主要动效语言：**轨迹线**。

- 轨迹线不是背景装饰，而是“一个想法如何抵达一次真实见面”的路径。
- 图片是轨迹中的事件节点，不是独立漂浮的素材。
- 点代表确认、相遇、信任或下一步。
- 轨迹颜色由青色到绿色，再在情绪节点出现少量洋红色。
- 每次滚动只推进一个主要叙事动作。

## 3. 页面体验蓝图

## 3.1 首页 `/`

### 核心命题

**让用户在 10 秒内理解：FitMeet 是关于真实见面，而不是另一款信息流社交产品。**

### 章节顺序

#### 01 / 首屏：一个世界正在形成

文案：

- Social World
- 让社交更简单
- 世界很大，别一个人抵达。
- 主按钮：了解社交助手如何辅助
- 次按钮：查看 FitMeet 应用

视觉：

- 保留轨迹线和九个图片碎片。
- 删除遮挡 `Social World` 中 O 字母的那一个碎片。
- 图片碎片保持不同景深，但标题始终是唯一主焦点。

动效：

- 轨迹线先出现，图片随后沿轨迹轻微进入。
- 标题按词组而非逐字长时间播放。
- 用户开始滚动后，碎片向轨迹节点收拢并退场。
- 首屏退场必须在下一张过渡图进入之前完成，禁止两个完整视觉层同时争夺。

#### 02 / 过渡：从想法进入真实场景

文案：

- 有些见面，只差一个人先开口。

视觉：

- 只替换此前重复出现的羽毛球大图。
- 使用新的蓝调城市公共空间过渡图。
- 图片承担首屏到 Social World 的承接，不新增第二组碎片。

动效：

- 轨迹线从首屏延伸进入图片。
- 图片由 88% 缩放到 100%，同时从低饱和逐渐恢复色彩。
- 文案在图片稳定后出现，不与图片放大同时启动。

#### 03 / Social World 预览：具体想做什么

文案：

- 想打球、去旅行，或者只是找人一起走走。
- 每一次见面，都从一个具体的想法开始。
- 按钮：进入完整的 Social World

视觉：

- 一个主场景、一条轨迹、四个场景索引。
- 不再连续播放四张全屏大图，避免与 `/world` 重复。

动效：

- 滚动只切换索引和局部预览，不进行四次完整全屏遮罩。
- 鼠标悬停场景索引时，主图焦点轻微移动，幅度不超过 3%。

#### 04 / 社交助手：把想法说清楚

文案：

- 你说想做什么，FitMeet 帮你把条件摆在眼前。
- 时间、地点、人数、预算和边界，都可以修改。
- 按钮：看看它怎么工作

视觉：

- 使用真实的产品交互演示，不使用抽象人物大图作为唯一内容。
- 展示一句输入、条件提取和确认计划三个状态。

动效：

- 输入内容随滚动逐步形成。
- 被识别的条件用轨迹点连接。
- 只有用户确认时，轨迹点才由空心变为实心。

#### 05 / 真实瞬间：结果而不是愿景

文案：

- 后来，他们真的见面了。
- 真实瞬间不是表演，是一次计划抵达生活之后留下的痕迹。
- 按钮：查看真实瞬间

视觉：

- 三张具有时间关系的真实场景：出发前 / 正在发生 / 结束之后。
- 图片说明包含时间、地点类型、活动和人数，不使用空泛情绪标签。

动效：

- 三张照片按时间顺序横向交接。
- 不使用叠卡无限堆积。

#### 06 / 安全：信任发生在见面之前

文案：

- 见面之前，把边界说清楚。
- 公共地点、明确计划、可修改的选择，以及随时退出的权利。
- 按钮：了解安全设计

动效：

- 轨迹在安全节点减速。
- 四项安全信息依次点亮，不使用警告式红色闪烁。

#### 07 / 应用入口

文案：

- 把确认好的计划，带进真实生活。
- FitMeet 应用即将上线。
- 按钮：获取上线消息

视觉与动效：

- 只展示真实产品状态与商店状态。
- 应用界面从轨迹末端出现，避免漂浮手机模型旋转。

## 3.2 Social World `/world`

### 核心命题

**这里不是第二个首页，而是一个可以探索的社交场景世界。**

### 章节顺序

1. **场景入口**  
   文案：今天，你想和别人一起做什么？
2. **自然语言探索**  
   输入示例：周六下午想找两三个人打羽毛球，最好离地铁近一点。
3. **场景地图**  
   运动 / 散步 / 旅行 / 共创 / 聚会。
4. **场景详情**  
   适合人数、时间、地点类型、计划难度和安全建议。
5. **告诉社交助手**  
   把当前场景和偏好带入 Agent。
6. **紧凑页脚**

### 文案规则

- 场景名称使用动作：“一起打球”“一起走走”“一起出发”。
- 不使用“探索无限可能”“重新连接世界”等抽象标题。
- 每个场景必须回答：做什么、和几个人、需要多久、在哪里更合适。

### 动效规则

- 参考地图探索，而不是连续电影式全屏滚动。
- 鼠标移动只影响局部景深与轨迹焦点。
- 筛选改变时，场景节点重新排序，时长 360–520ms。
- 所有场景都可键盘访问，动效不能成为唯一导航方式。

## 3.3 真实瞬间 `/moments`

### 核心命题

**让用户看见“计划真的进入了生活”。**

### 章节顺序

1. **首屏**  
   标题：后来，他们真的见面了。
2. **时间切片**  
   出发前 / 正在发生 / 结束之后。
3. **真实故事**  
   每个故事包含需求、计划、现场和一句真实感受。
4. **场景索引**  
   运动 / 旅行 / 城市 / 兴趣 / 陪伴。
5. **下一步入口**  
   进入 Social World / 告诉社交助手。
6. **紧凑页脚**

### 文案示例

- 原本只是想找个人打球，后来固定在每周四见面。
- 三个人，一条临时决定的路线，最后走到了天黑。
- 没有盛大的故事，只是那天没有一个人吃饭。

### 动效规则

- 像翻看真实影像档案，而不是品牌广告片。
- 图片之间保留停顿，禁止自动高速轮播。
- 文字出现应晚于图片 160–240ms，让用户先看见人。

## 3.4 社交助手 `/agent`

### 核心命题

**把一句想见面的想法，整理成一份可以确认的计划。**

### 章节顺序

1. **首屏产品演示**  
   一句话输入 + 条件提取 + 计划结果同屏出现。
2. **一次完整任务**  
   从“周六想打球”到确认时间、地点、人数和预算。
3. **它如何辅助**  
   听懂想法 / 标出不确定 / 给出建议 / 等待确认。
4. **它不会替你做什么**  
   不替你发邀请、不替你承诺、不替你忽略边界。
5. **修改与确认**  
   展示真实的改时间、换地点和取消计划状态。
6. **应用入口**
7. **紧凑页脚**

### 首屏文案

- 从一句想法，到一份可以确认的计划。
- FitMeet 帮你整理时间、地点、人数、预算和边界。邀请谁、要不要见面，仍由你决定。
- 按钮：试着整理一个计划

### 产品演示文案

- 输入：周六下午想找两三个人打羽毛球，预算每人 60 元以内。
- 已识别：周六下午 / 2–3 人 / 羽毛球 / ¥60 内。
- 需要确认：你更希望离家近，还是离地铁近？
- 计划建议：周六 15:00，公共运动场馆，预计 90 分钟。

### 动效规则

- 产品状态由用户动作触发，不自动无限循环。
- 条件提取使用高亮和轨迹连接，不使用“AI 粒子爆炸”。
- 不确定项使用空心节点，已确认项使用实心节点。
- 状态切换 240–420ms；输入反馈低于 150ms。
- 必须支持减少动态效果设置。

## 3.5 安全设计 `/safety`

### 核心命题

**安全不是一页承诺，而是每一步都能被用户看见的设计。**

### 章节顺序

1. 首屏：见面之前，把边界说清楚。
2. 计划透明：时间、地点、参与者和活动内容。
3. 用户确认：建议可以修改，邀请需要确认。
4. 公共空间建议：优先公共、熟悉、可离开的场所。
5. 控制权：拒绝、退出、屏蔽和举报。
6. 实际支持入口：支持中心 / 举报与求助。
7. 社区准则入口。
8. 紧凑页脚。

### 动效规则

- 低频、稳定、明确。
- 不使用快速闪烁、强烈警示色和制造恐惧的画面。
- 安全半径用缓慢扩散的圆表达，扩散周期不少于 2.4 秒。

## 3.6 FitMeet 应用 `/app`

### 核心命题

**网页让人理解 FitMeet，应用让计划真正发生。**

### 章节顺序

1. 首屏：把确认好的计划，带进真实生活。
2. 三个核心界面：提出想法 / 确认计划 / 见面提醒。
3. 产品流程：想法—计划—见面。
4. 安全与控制：修改、退出、举报。
5. 上线状态：苹果应用商店 / 谷歌应用商店。
6. 获取上线消息。
7. 紧凑页脚。

### 文案规则

- 未上线功能不得写成已经可用。
- 清楚区分“即将上线”“设计中”“计划支持”。
- CTA 只使用“获取上线消息”，不使用虚假的“立即下载”。

### 动效规则

- 产品界面使用真实状态过渡。
- 禁止手机模型持续自转。
- 每个功能展示不超过 8 秒，并允许手动切换。

## 4. 文案系统

### 文案四层结构

1. **事实**：用户正在做什么。
2. **困难**：哪里麻烦、哪里不确定。
3. **FitMeet 的帮助**：具体整理什么、展示什么。
4. **控制权**：谁确认、谁决定、如何退出。

### 推荐句式

- 想做什么，先说出来。
- 不确定的地方，留给你确认。
- 把条件摆在眼前，再决定要不要出发。
- 有些见面，只差一个人先开口。
- 后来，他们真的见面了。
- 世界很大，别一个人抵达。

### 禁用句式

- 重新定义社交。
- 打破边界，连接无限可能。
- 开启全新的社交体验。
- 让每一次相遇更有意义。
- AI 赋能下一代社交生态。

除非后面立即给出具体事实，否则不使用上述抽象表达。

## 5. 动效系统

### 动效层级

#### A / 品牌动效

- 轨迹线绘制、延伸、减速、连接。
- 仅用于首屏、章节交接和关键确认。

#### B / 叙事动效

- 图片进入、焦点移动、场景交接。
- 每次滚动最多一个主变化和一个辅变化。

#### C / 产品反馈

- 输入、高亮、确认、错误、修改、退出。
- 速度优先，控制在 120–420ms。

#### D / 基础界面

- 导航、按钮、下拉、链接和手风琴。
- 统一 180–260ms。

### 滚动规则

- 禁止连续多个全屏 sticky 章节互相覆盖。
- 同一元素不重复进入两次。
- 上一章节的主要图片必须在下一章节主图进入前完成退场。
- 章节间至少保留 12–18vh 的视觉呼吸区。
- 任何固定层都必须定义进入、保持和退场三个状态。

### 性能规则

- 首屏优先加载一张主图和必要品牌碎片。
- 非首屏图片按接近视口的顺序加载。
- 同时运行的图片缩放/遮罩动画不超过两个。
- transform 与 opacity 优先，避免滚动时高频修改布局属性。
- 所有页面支持 `prefers-reduced-motion`。
- 桌面端目标：LCP 小于 2.5 秒，动画主线程长任务低于 50ms。

## 6. 视觉系统约束

### 色彩

- 纸张色：`#F4F1EC`
- 墨色：`#08090B`
- 夜色：`#050607`
- 青色：`#08E4F2`
- 轨迹绿：`#28F0C0`
- 情绪洋红：`#F14DB2`

原则：每个屏幕最多一个高饱和主色焦点，青、绿、洋红不得同时大面积竞争。

### 字体

- 中文正文：PingFang SC 优先。
- 英文品牌标题：Arial / Avenir Next / SF Pro Display 体系。
- 只保留英文 `Social World` 与 `FitMeet`。
- 中文大标题使用较紧字距，但避免低于 `-0.06em`。

### 桌面端字号建议

- 品牌首屏：96–160px。
- 页面首屏标题：64–96px。
- 章节标题：48–72px。
- 卡片标题：22–32px。
- 正文：16–20px。
- 标签与元信息：12–14px。

### 图片原则

- 优先真实人物、真实活动、公共空间和具体行动。
- 避免人物摆拍直视镜头。
- 避免同一活动在首页和内页重复作为大图。
- 每张大图都必须有叙事角色：起点、过渡、证明、边界或结果。

## 8. 实施优先级与验收标准

### P0 / 叙事骨架

- 首页不存在重复羽毛球大图。
- 新过渡图能够从首屏轨迹自然承接到 Social World 预览。
- Agent 首屏可以完成一次从输入到确认的任务。
- 滚动过程中不出现两个主标题或两张主图同时竞争。

### P1 / 页面职责

- 首页负责品牌总览。
- World 负责场景探索。
- Moments 负责真实证明。
- Agent 负责产品能力。
- Safety 负责信任机制。
- App 负责上线转化。

### P2 / 一致性

- 所有页面使用相同导航与紧凑页脚。
- 所有页面文案能回答“用户正在做什么”。
- 动效只使用轨迹、图片交接和产品反馈三类主要语言。

### P3 / 获奖级精修

- 逐屏检查视觉焦点、图片裁切、文字换行、间距和节奏。
- 对每个 sticky 章节录制滚动视频并逐帧检查进入与退场。
- 桌面端 1440×900、1680×1050、1920×1080 三个视口完成验收。
- 类型检查和生产构建通过。
