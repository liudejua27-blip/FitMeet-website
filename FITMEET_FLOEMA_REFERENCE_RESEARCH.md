# Floema 二级页面研究与 FitMeet 转译 V1.0

研究日期：2026-07-10

主要参考：

- [Floema Home](https://www.floema.com/)
- [Floema Products](https://www.floema.com/produtos)
- [Floema About](https://www.floema.com/sobre)
- [Floema Sustainability](https://www.floema.com/sustentabilidade)
- [Floema Journal](https://www.floema.com/jornal)
- [Floema Product Detail](https://www.floema.com/produtos/urban/espreguicadeira-sun-lounger-plaza)
- [Floema Case Study](https://www.floema.com/jornal/casos-de-estudo/trilho-cangosta-do-estevao)

本研究只提炼信息架构、空间节奏、摄影优先级和交互语法，不复制 Floema 的素材、文案、产品分类和坐标。

## 1. Home

### 观察

- 中心宣言被不同尺寸和清晰度的影像包围。
- 滚动不是进入下一个普通 Section，而是选择其中一个影像坐标。
- 小影像扩张为全屏城市或自然章节。
- 每章保持编号、类别、短说明和一个行动。

### FitMeet 转译

- 首页使用 Social Orbit 星图。
- 碎片不是产品与材料，而是运动、旅行、露营和城市活动邀请。
- 光谱轨迹负责连接分散的社交意愿。

## 2. Products

### 观察

- Products 页从沉浸叙事切换为高密度探索。
- 顶部提供 Collection 与 Product 类型筛选。
- 大量产品以统一的信息规则排列，分类、颜色和规格作为次级信息。
- 页面任务改变时，结构也改变；没有机械复制首页的全屏章节。

### FitMeet 转译

- World 可以借鉴“可探索集合”，但不使用传统商品卡片网格。
- 运动、旅行、户外和城市兴趣成为地图坐标与空间入口。
- 场景筛选可以存在，但必须附着在地图或轨迹上，而非悬浮筛选卡片。

## 3. About

### 观察

- 首屏使用品牌宣言和大比例真实摄影。
- 品牌起点由一个具体问题推动，而不是空泛愿景。
- 历史使用可拖拽横向时间轴，年份、图片和事件交错。
- 后段使用“设计—建造—实施”的过程结构，再进入原则和团队。

### FitMeet 转译

- About 从“想做一件事却找不到同路的人”这一具体问题开始。
- FitMeet 当前没有可公开验证的公司历史，因此不伪造年份时间轴。
- 使用 `NOW / NEXT / LATER` 代替历史年份。
- 过程结构改为“说出—理解—确认—发生”，但避免与 Agent 页重复完整产品演示。

## 4. Sustainability

### 观察

- 页面先表达明确立场，再解释材料和技术。
- 材料不以图标卡片介绍，而通过真实近景、参数与项目结果说明。
- 项目案例把抽象价值重新放回真实空间。
- 末尾以三项支柱收束。

### FitMeet 转译

- Safety 先表达“见面之前，边界已经开始”，再解释机制。
- 身份、意愿、公共地点、真实经历和异常反馈对应五层信任半径。
- 每层配真实公共空间与行为细节，不使用盾牌图标墙。
- 未实现机制标记 `BUILDING`，不伪装为当前保证。

## 5. Journal

### 观察

- Journal 具有明确编辑立场和类别索引。
- Featured Case Studies 使用大摄影、摘要和单一阅读行动。
- 普通内容以日期、栏目和标题建立节奏。
- 详情页以阅读时间、画廊、长文、项目参与方和材料信息形成完整证据链。

### FitMeet 转译

- Moments 是动态品牌杂志，不是社交 Feed。
- 每个故事围绕时间、地点、共同动作和结果展开。
- 故事详情使用环境—行动—细节—结果的影像顺序。
- 人物和影像必须授权；没有真实内容时只显示品牌预告。

## 6. Product Detail

### 观察

- 单一对象占据绝大部分视觉注意力。
- 文案克制，规格和选项在产品被理解后出现。
- 相关产品位于页面末端，不抢首屏。

### FitMeet 转译

- Agent 页把“一句需求”当作单一产品对象。
- 时间、地点、人数、兴趣、预算和边界作为规格节点逐步出现。
- UI 在需求被理解之后才展开。
- App 页把需求、理解、调整、确认、发生作为产品状态，而不是功能卡片。

## 7. Case Study

### 观察

- 详情页先给完整地点与项目氛围，再进入技术解释。
- 长文与图片交替，不把所有信息压缩为摘要卡片。
- 项目角色和材料在末端形成可信细节。

### FitMeet 转译

- Moments 故事先给地点和共同经历，再解释如何发生。
- 只有真实授权故事可以提供人物、时间和结果。
- Agent 不应成为故事主角；它只作为促成真实行动的背景机制。

## 8. 最终采用原则

1. 页面结构由内容任务决定，不使用同一模板复制六次。
2. 摄影先于文案，场景先于功能。
3. 单屏只保留一个主要标题和一个主要行动。
4. 真实案例必须提供事实与授权，不制造社会证明。
5. 产品能力与未来愿景必须分层。
6. 导航与品牌信号保持稳定，页面创意可以变化。
7. 页面之间通过当前影像、胶片、节点、半径或地平线连续过渡。
