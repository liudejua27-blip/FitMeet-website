# FitMeet Social World Design System V5.0

本文件是全站最高设计合同。首页已确定的核心概念是 **Social Orbit**：每一张影像都是一个尚未发生的社交邀请，Agent 让分散的邀请形成轨道。

二级页面与首页必须属于同一个品牌世界，但不得重复首页的影像星图。每页使用独立的空间隐喻解释自己的内容任务。

## 1. 品牌表达

- 英文主标题：`SOCIAL WORLD`
- 中文主张：`让社交更简单`
- 情绪句：`世界很大，别一个人抵达。`
- 产品转场：`你说一句，世界开始组织。`
- 长期愿景：`让人与人之间，少一点寻找，多一点抵达。`
- 产品边界：`Agent 让社交更轻，关系仍由人建立。`
- 最终转化：`让下一次出发，不再等一个碰巧。`

FitMeet 当前是辅助社交决策层。Agent 可以理解需求、整理条件、生成计划和降低沟通成本，但发布、邀请、确认和见面等关键动作必须由用户决定。

## 2. Floema 借鉴边界

### 可借鉴

- 真实摄影先于说明文案。
- 小图靠近并扩张为完整空间。
- 固定、克制、稳定的胶囊导航。
- 页面根据内容任务改变结构，而不是复制同一套 Section。
- About 的时间轴、过程与人物叙事。
- Sustainability 的宣言、材料、案例、原则结构。
- Journal 的编辑索引与案例长文结构。
- Product Detail 的单一对象、规格与相关内容节奏。
- 长缓动、低振幅视差、图片先到位、文字后出现。

### 必须原创

- FitMeet 场景、人物、摄影和品牌视频。
- 中文与英文文案。
- Social Orbit 路径、节点、光谱脉冲和 Logo 曲线蒙版。
- Agent 的需求理解和确认交互。
- World、Moments、Safety、About 与 App 的页面空间隐喻。
- App Store / Google Play 上线舱。
- 从现实进入产品、再返回现实的闭环。

不得复制 Floema 的素材、精确坐标、橙黄识别色、产品分类、团队信息或商业文案。

## 3. 全站空间模型

全站存在一个持续的 `SpatialShell`：

- 固定品牌导航在全部品牌页持续存在。
- 页面离场时，当前主要影像或轨迹成为转场蒙版。
- 新页面不从白屏或黑屏重新开始，而从同一空间缩放、平移或曲线揭示。
- 页面正文可以使用标准文档流，但章节边界必须由摄影、空间位移或光线变化衔接。
- 法务页不使用沉浸式固定卷轴，只保留品牌导航、排版和轻量轨迹。

## 4. 页面创意矩阵

| Page | Spatial metaphor | Dominant motion | Content role |
| --- | --- | --- | --- |
| Home | Social Orbit 影像星图 | fragment zoom | 建立向往 |
| World | 无限场景地图 | multi-axis pan / portal zoom | 展示活动世界 |
| Moments | 时间胶片与记忆档案 | film rail / aperture reveal | 真实经历与情感证明 |
| Agent | 一句话形成轨道 | semantic node extraction | 解释辅助社交产品 |
| Safety | 透明边界与信任半径 | concentric reveal | 降低陌生社交顾虑 |
| About | 从一个节点扩展到世界 | draggable timeline / horizon expansion | 使命与发展路径 |
| App | 上线舱与界面轨道 | interface choreography | 产品理解与转化 |
| Guidelines / Legal | 编辑型品牌文档 | subtle line reveal | 规则、隐私与协议 |

## 5. 色彩

| Token | Value | 用途 |
| --- | --- | --- |
| `--paper` | `#F4F1EC` | 暖白品牌空间 |
| `--paper-bright` | `#F8F6F2` | Safety 与规则页 |
| `--ink` | `#08090B` | 文字、导航与深色界面 |
| `--night` | `#050607` | Agent、App 与最终章节 |
| `--slate` | `#182025` | 深色次级表面 |
| `--cyan` | `#08E4F2` | 路径、时间与提示 |
| `--aqua` | `#28F0C0` | 在线、完成与可信状态 |
| `--magenta` | `#F14DB2` | 轨迹尾部与确认强调 |
| `--line-light` | `rgba(8,9,11,.14)` | 纸面结构线 |
| `--line-dark` | `rgba(255,255,255,.16)` | 深色结构线 |

禁止大面积通用 AI 渐变。光谱只能作为窄路径、节点、按钮焦点、进度、可信状态和确认反馈。

页面可根据内容改变明暗节奏，但不得为每页重新发明一套品牌色。

## 6. 排版

- 英文展示标题：无衬线、宽字面、紧字距；桌面 `72–132px`。
- 中文展示标题：短句、紧行距、负字距；桌面 `56–104px`。
- 正文：`16–22px`，最大行宽约 `32–40` 个汉字。
- 工具文字：`0.58–0.74rem`，大写英文与高字距。
- 数字、时间、章节编号使用等宽或近似等宽节奏。
- 法务正文 `16–18px`，行高不低于 `1.75`。
- 所有按钮、输入、筛选、标签和导航必须显式定义字体。

禁止在每个 Hero 上方机械增加 eyebrow、badge 或说明胶囊。只有已批准的工具标签、章节编号和功能状态可以使用小型大写英文。

## 7. 摄影与视频

### 页面专属摄影

每个一级品牌页必须拥有独立 Hero Master 和镜头语言。统一调色不等于可以复用全屏摄影；素材源、页面分配、生成提示词与重复图片验收以 `FITMEET_ASSET_DIRECTION.md` 为准。

全站应像同一个世界由不同摄影机观察：Home 发现邀请，World 穿越领地，Moments 推进记忆胶片，Agent 追踪一句话穿过交通大厅，Safety 在公共空间揭示信任景深，About 观察共同行动生长为网络，App 使用受控产品电影语言。

只替换文案但继续使用同一张 Hero 图片，不构成独立页面设计。

### 总体比例

- 70% 城市、旅行与户外纪实大场景。
- 20% 夜间电影感活动场景。
- 10% 手、装备、地图、光线、集合点等细节。

### 规则

- 人物处于真实行动中，不集体看镜头。
- 自然肤色、统一电影调色、轻微颗粒。
- 优先展现地点和共同动作，再展示产品。
- 不使用头像墙、摆拍合影、水印、虚构合作 Logo 或占位图片。
- Moments 只使用取得授权的真实人物故事；概念摄影不能冒充用户证言。
- About 不使用虚构创始人、团队人数、办公室或发展年份。
- 视频不得作为首页 LCP；进入视口才播放，离开后暂停。

## 8. 共享组件

### Global Navigation

- 左：FitMeet Logo + `FITMEET`，链接 `/`。
- 中：`WORLD / MOMENTS / AGENT / SAFETY`。
- 右：`APP · 即将上线`。
- 当前页面以细光谱底线、文字状态或胶囊内部节点表示，不改变胶囊尺寸。
- 导航根据背景明暗改变文字与表面颜色。

### Global Footer

- 品牌：`FITMEET / SOCIAL WORLD`。
- 句子：`让人与人之间，少一点寻找，多一点抵达。`
- 主链接：World、Moments、Agent、Safety、About、App。
- 规则链接：Community Guidelines、Privacy、Terms。
- 底部：`© 2026 FITMEET / DESIGNED FOR REAL CONNECTIONS`。

### Store Capsule

- App Store 与 Google Play 共享一个连续外框。
- 当前只显示 `即将上线`，不得链接虚构地址。
- 真实商店地址上线后才替换为外部链接。

### Section Index

- 章节编号固定为两位数：`01`、`02`。
- 类别标签不超过一行。
- 单屏最多一个主要标题和一个主要 CTA。

## 9. 页面专属规则

### World

- 首屏是一张可探索的无限场景地图，不是分类卡片网格。
- 四个当前场景为 Move、Go、Outdoor、Create。
- `Social Life` 必须标记为 `NEXT`，不能与当前能力同级展示。
- 主要摄影占每个场景至少 80% 视口。

### Moments

- 结构接近编辑杂志与案例档案，不做社交 Feed。
- 故事必须包含时间、地点、共同动作和结果，避免空泛评价。
- 未获得真实授权前显示 `STORIES ARE BEING MADE`，不发布虚构人物证言。

### Agent

- 需求句是页面主角，产品 UI 在语义节点形成后才出现。
- 页面必须明确 `ASSISTED, NOT REPLACED`。
- 生成结果可以本地模拟，但不能暗示真实匹配已经发生。
- 确认按钮必须更新真实界面状态。

### Safety

- 安全通过身份、意愿、地点、真实经历和反馈五层半径表达。
- 未上线能力使用 `逐步建立` 或 `设计原则`，不得写成已经实施的保证。
- 不使用盾牌图标墙或绿色合规卡片墙。

### About

- 不伪造公司历史、融资、团队、办公室或用户规模。
- 采用 `NOW / NEXT / LATER` 表达真实发展路径。
- 可展示品牌使命、问题起点、产品原则和未来愿景。

### App

- 展示需求、理解、调整、确认、发生五个产品状态。
- App UI 文本和控件必须代码原生，不把截图当成交互界面。
- 当前不得出现二维码、评分、下载量或虚构商店评论。

### Guidelines / Privacy / Terms

- 保持编辑型长文排版与清晰目录。
- Community Guidelines 可以使用正式品牌文案。
- Privacy 与 Terms 的法务正文必须由专业法律人员审核；设计稿只提供结构和非约束性导语。

## 10. 动效

- Lenis 只在精细指针与非 Reduced Motion 环境启用。
- 主要动画使用 transform、opacity、clip-path 与 SVG 路径。
- 图片先稳定，文字延迟约 `200–350ms`。
- 中文逐字显示 `24–35ms / 字`。
- 视差幅度 `4%–10%`，避免漂浮模板感。
- 页面转场 `650–1000ms`，使用 Logo 曲线蒙版、影像扩张或轨迹聚合。
- 磁吸只用于导航主动作、商店舱、场景 CTA 和 Agent 确认按钮。
- 横向时间轴或胶片使用垂直滚动驱动，内部水平动画必须线性映射。
- 不动画被 ScrollTrigger 固定的容器本身，只动画内部子层。
- 所有动画在卸载时清理。

## 11. 无障碍与性能

- 语义导航、主标题、按钮、表单标签和媒体替代文本完整。
- 逐字动画保留整句可访问名称，视觉字符对辅助技术隐藏。
- 所有交互可键盘访问，并具有清晰 focus-visible 状态。
- Reduced Motion：取消 Lenis、pin、scrub、视差、路径绘制与视频自动播放，恢复普通文档流。
- 首屏优先静态图片；下游视频 `preload="none"` 并由可见性控制。
- Next Image 使用准确 `sizes`；不可见大图保持延迟加载。
- 页面切换后清理旧 ScrollTrigger 和媒体播放状态。

## 12. 桌面验收

- `1440 × 1000`
- `1680 × 1050`
- `1920 × 1080`

全部页面必须满足：

- 无横向溢出、标题裁切、白屏和硬色带。
- 固定导航在明暗背景上保持可读。
- 透明重叠层不阻断底层交互。
- 页面首屏在三秒内表达本页任务。
- 从任何品牌页进入其他页面都不出现裸露白屏跳转。
- 每页至少有一个与该页专属空间创意一致的主要动效。
- Reduced Motion 信息完整。
- Browser 控制台无应用 error/warn。
