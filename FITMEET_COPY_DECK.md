## 2. 文案禁止项

现阶段禁止：

- `Agent 自动替你交朋友`
- `全自动完成社交`
- `100% 精准匹配`
- `绝对安全`
- `AI 替代人与人的沟通`
- `已经覆盖所有生活服务`
- 未经证实的用户规模、成功率、评分、商店评价和合作伙伴

统一产品表达：

> FitMeet 是辅助社交决策层。它理解需求、整理条件、减少寻找和沟通成本，关键行动始终由用户确认。

## 5. Agent

### Hero

```text
YOUR SOCIAL AGENT

你负责想去，
Agent 负责把条件整理清楚。

它不会替你交朋友，
也不会在未经确认的情况下替你行动。

它理解需求、减少寻找和沟通消耗，
让真实的人更容易见面。

看看 Agent 如何工作
```

### 01 Say It

```text
SAY IT

像和朋友说话一样，
告诉 Agent 你想做什么。

不用研究复杂分类，
也不用在无数群聊和帖子里碰运气。
```

### 02 Understand

```text
UNDERSTAND

一句需求背后，
还有时间、地点、人数、兴趣和边界。

Agent 将这些条件整理成一份
清晰、可调整、可确认的社交计划。

TIME / PLACE / INTEREST / PEOPLE / BUDGET / BOUNDARY
```

### 03 Connect

```text
CONNECT

计划由 Agent 整理，
决定始终由你完成。

在邀请、发布、确认和见面之前，
你都可以重新选择。
```

## 6. Safety

### Hero

```text
TRUST, BEFORE MEETING

见面之前，
边界已经开始。

社交可以自然发生，
但身份、意愿、地点和行为规则必须足够清晰。
```

### Identity

```text
01 / IDENTITY

真实身份，
不等于公开隐私。

必要的验证用于帮助平台判断真实，
不意味着把敏感信息暴露给陌生人。
```

### Consent

```text
02 / CONSENT

每一步，
都需要你的确认。

Agent 可以给出建议、整理计划，
但不会越过你的意愿替你作出关键决定。
```

### Public First

```text
03 / PUBLIC FIRST

第一次见面，
优先从公共空间开始。

清晰的时间、集合地点和活动安排，
让见面双方都知道将要发生什么。
```

### Real Experience

```text
04 / REAL EXPERIENCE

评价属于真实经历之后，
不是见面之前的包装。

未来的信任体系应围绕真实参与、
履约情况与社区行为逐步建立。
```

### Response

```text
05 / RESPONSE

异常应该被看见，
反馈应该得到处理。

骚扰、虚假身份、临时失联和不当行为，
都需要清晰的举报入口和处理机制。
```

### Data

```text
YOUR DATA, WITH A PURPOSE

产品设计遵循：
最少必要、用途明确、用户可控。

具体的数据收集、保存和删除规则，
以正式隐私政策为准。
```

结尾：

```text
安全不是阻止关系开始，
而是让每个人都能更安心地作出选择。

阅读社区准则
```

## 9. Community Guidelines

```text
COMMUNITY GUIDELINES

自由连接，
建立在相互尊重之上。

01 真实表达自己的身份和需求
02 尊重对方的选择与拒绝
03 在行动前说清时间、地点和费用

05 不进行骚扰、歧视、欺骗或诱导
06 遇到异常情况及时结束并反馈

每一次认真对待，
都在帮助 Social World 变得更可信。
```
